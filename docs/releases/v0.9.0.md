# Release v0.9.0 — DTG/OpenVTC/VTI Compatibility Profiles and Runtime Trust Artifact Alignment

**Release date:** 2026-05-06  
**Status:** Ready for adoption review

## Summary

TIS v0.9.0 turns related-work references to DTG Credentials, OpenVTC, and Verifiable Trust Infrastructure into executable compatibility profiles. The release adds schemas, examples, documentation, and a drift-tracking matrix so implementation changes in those repos can be evaluated against TIS artifact contracts.

## Added

- DTG VC envelope compatibility schema and credential subject profile.
- DTG personhood membership overlay and relationship-card profile.
- OpenVTC runtime profile schemas for relationship state, VRC issuance receipts, configuration evidence, and DIDComm routing evidence.
- VTI runtime assurance schemas for VTA context, ACL entries, VTA authorization credentials, sealed transfer envelopes, DID template references, TEE attestation references, and provision-integration receipts.
- Machine-readable cross-repo compatibility matrix and schema.
- Cross-repo examples under `examples/cross-repo/dtg-openvtc-vti/`.
- Upstream reference documentation for:
  - <https://github.com/OpenVTC/dtg-credentials>
  - <https://github.com/OpenVTC/openvtc>
  - <https://github.com/OpenVTC/verifiable-trust-infrastructure>

## Changed

- Refreshed README and documentation index for v0.9.0.
- Updated validation coverage manifest to include DTG/OpenVTC/VTI profile artifacts.
- Expanded validation tooling to check the new profile examples and compatibility matrix.
- Clarified that TIS does not own upstream implementation semantics; it defines executable artifact contracts for assurance, auditability, and interoperability.

## Fixed

- Removed stale current-release references to earlier releases from active documentation.
- Added explicit drift-review triggers for upstream implementation changes.

## Assurance impact

This release improves evidence quality across authority, delegation, scope, enforcement, revocation, and auditability. It allows implementers to connect credential issuance, runtime relationship workflows, VTA authorization, sealed transfer, and attestation evidence to TIS decision receipts and registry artifacts.
