# Release v0.8.0 — TSMM Alignment Profile and Cross-Repo Artifact Interoperability

**Release date:** 2026-05-05  
**Status:** Additive harmonization release

## Executive summary

Trust Infrastructure Schemas v0.8.0 establishes a formal interoperability profile with the Trust Systems Meta Model (TSMM). The release clarifies that TSMM owns semantic modeling while this repository owns executable artifact contracts.

## What changed

- Added `model/tsmm-artifact-mapping.schema.json` and `model/tsmm-artifact-mapping.json`.
- Added TSMM alignment documentation across decision receipts, authority boundaries, evidence, registries, and executable artifact walkthroughs.
- Added TSMM-derived authority boundary, evidence bundle, evaluation envelope, decision receipt, and registry entry examples.
- Added a complete cross-repo composition pack under `examples/composition/tsmm-aligned/`.
- Updated validation coverage and local validation script to include the TSMM mapping and composition examples.
- Updated schema `$id` values and schema URI references for v0.8.0.
- Removed stale local-only `.DS_Store` artifact.

## Governance impact

The release makes the repository more adoption-ready for teams that need to move from governance models to executable assurance artifacts. A TSMM-modeled trust decision can now be represented as a TIS decision receipt with explicit authority, evidence, assurance, policy, result, and effect constraints.
