# v0.10.0 — TSMM-Aligned Runtime Assurance Artifact Contracts

Released: 2026-06-26

## Release thesis

TIS v0.10.0 makes TSMM runtime governance executable. The release establishes TSMM as the semantic governance model and TIS as the machine-validatable artifact contract layer for runtime assurance.

## Added

- Added TSMM runtime governance projection schema and validated example.
- Added Trust Task reference, manifest-reference, lifecycle-event, and execution-receipt schemas.
- Added TSMM-aligned Trust Task lifecycle vocabulary separating lifecycle state, decision outcome, and effect admission.
- Added VTI authorization evidence alignment for implementation roles, capability grants, allowed contexts, consumer kind, lifecycle state, step-up metadata, and TSMM authority references.
- Added OpenVTC state-normalization and task-evidence schemas.
- Added evidence bundle integrity profile with canonicalization, bundle digest, detached proof, and signature metadata.
- Added VTI status-list reference profile for revocation, suspension, expiry, and activation evidence.
- Added registry publication profile with assurance-level evidence requirements and explicit discovery-versus-authorization boundary.
- Added TSMM runtime governance and Trust Task crosswalk documentation.
- Added v0.10.0 cross-repo runtime assurance example pack.

## Changed

- Updated release metadata from v0.9.0 to v0.10.0.
- Updated VTI ACL role vocabulary to align with implementation roles: `admin`, `initiator`, `application`, `reader`, and `monitor`.
- Reframed VTI ACL entries as authority evidence rather than generic permission records.
- Updated cross-repo compatibility tracking to include TSMM as the semantic upstream and runtime assurance projection as the binding pattern.
- Expanded validation coverage for all new schemas and examples.

## Fixed

- Removed local `.DS_Store` artifact that caused repository hygiene lint failure.
- Updated package metadata and release documentation so `npm run check` has a clean v0.10.0 release target.
