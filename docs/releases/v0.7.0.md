# v0.7.0 — Executable Governance Contract Pack and Validation Hardening

**Release date:** 2026-05-05

## Summary

`v0.7.0` strengthens `trust-infrastructure-schemas` as a canonical machine-readable trust artifact layer. It introduces decision receipts, hardens authority-boundary semantics, expands validation coverage across all major artifact families, and refreshes documentation for cross-repo adoption.

## Why this release matters

Trust infrastructure cannot stop at schema-valid payloads. Operational trust decisions require evidence of what was evaluated, which authority boundary applied, what policy was used, what decision was made, and what conditions constrain reliance. This release adds that missing decision-evidence layer and makes repository validation coverage explicit.

## Compatibility

This release is designed as a minor-version-compatible expansion. New fields are optional unless introduced in new artifact families. Existing credential, assurance, conformance, controls, and registry consumers should continue to work, while implementers can incrementally adopt decision receipts and richer authority boundaries.

## Migration guidance

Consumers should:

1. update pinned schema references to `v0.7.0` when adopting new artifacts;
2. validate examples with `npm run check`;
3. adopt decision receipts where relying-party decisions need auditable evidence;
4. preserve authority-boundary metadata with declarations, evidence, evaluations, and decisions;
5. use `validation/artifact-coverage.json` to understand which artifact families are covered by CI.
